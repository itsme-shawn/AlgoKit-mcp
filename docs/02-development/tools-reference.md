## search_problems

### 설명
다양한 필터 옵션을 사용하여 백준 온라인 저지(BOJ) 문제를 검색합니다.
키워드, 난이도(티어), 알고리즘 태그 등으로 원하는 문제를 찾을 수 있습니다.

### 입력 스키마

```typescript
{
  query?: string;           // 검색어 (제목, 문제 번호, 태그)
  level_min?: number;       // 최소 난이도 (1-30)
  level_max?: number;       // 최대 난이도 (1-30)
  tag?: string;             // 알고리즘 태그 키
  sort?: "level" | "id" | "average_try";  // 정렬 기준
  direction?: "asc" | "desc";             // 정렬 방향
  page?: number;            // 페이지 번호 (기본값: 1)
}
```

#### 파라미터 상세

| 파라미터 | 타입 | 필수 | 기본값 | 설명 |
|---------|------|------|--------|------|
| `query` | string | 선택 | - | 검색어 (예: "이분 탐색", "1927") |
| `level_min` | number | 선택 | 1 | 최소 난이도 (1=Bronze V, 30=Ruby I) |
| `level_max` | number | 선택 | 30 | 최대 난이도 |
| `tag` | string | 선택 | - | 태그 키 (예: "dp", "greedy") |
| `sort` | string | 선택 | "level" | 정렬 기준 |
| `direction` | string | 선택 | "asc" | 정렬 방향 |
| `page` | number | 선택 | 1 | 페이지 번호 (한 페이지당 50개) |

### 출력 스키마

```typescript
{
  count: number;            // 총 검색 결과 개수
  page: number;             // 현재 페이지
  problems: [
    {
      problemId: number;
      title: string;        // 한글 제목
      level: number;        // 1-30
      tierName: string;     // "Gold III"
      tierBadge: string;    // "🟡 Gold III"
      tags: string[];       // ["DP", "그리디"]
      acceptedCount: number;
      averageTries: number;
      bojLink: string;
    }
  ]
}
```

### 사용 예시

#### 예시 1: 키워드 검색
**사용자**: "이분 탐색 문제 찾아줘"

**도구 호출**:
```json
{
  "query": "이분 탐색"
}
```

**응답**:
```
총 89개의 이분 탐색 문제를 찾았습니다:

1. [1920] 수 찾기 (Silver IV)
   - 태그: 이분 탐색, 자료 구조
   - 89,234명 해결, 평균 1.8회 시도

2. [1654] 랜선 자르기 (Silver II)
   - 태그: 이분 탐색, 매개변수 탐색
   - 67,892명 해결, 평균 2.4회 시도

...
```

#### 예시 2: 난이도 범위 필터링
**사용자**: "Gold 티어의 DP 문제 중에서 쉬운 순으로 보여줘"

**도구 호출**:
```json
{
  "level_min": 11,
  "level_max": 15,
  "tag": "dp",
  "sort": "level",
  "direction": "asc"
}
```

#### 예시 3: 특정 문제 번호 검색
**사용자**: "1927번 문제 있어?"

**도구 호출**:
```json
{
  "query": "1927"
}
```

### 주의사항

- `level_min`과 `level_max`는 1-30 범위여야 합니다
- `level_min`이 `level_max`보다 크면 에러 발생
- `page`는 1부터 시작합니다
- 검색 결과가 없으면 빈 배열 반환

### 에러 케이스

| 에러 | 원인 | 해결 방법 |
|------|------|----------|
| "Invalid level range" | level_min > level_max | 레벨 범위 확인 |
| "Invalid tag key" | 존재하지 않는 태그 | `search_tags`로 유효한 태그 확인 |
| "Page out of range" | 페이지 범위 초과 | 1부터 시작하는 유효한 페이지 입력 |

## get_problem

### 설명
특정 BOJ 문제의 상세 메타데이터를 조회합니다.
문제 번호만 있으면 제목, 난이도, 태그, 통계 등 모든 정보를 얻을 수 있습니다.

### 입력 스키마

```typescript
{
  problem_id: number;       // BOJ 문제 번호 (필수)
}
```

#### 파라미터 상세

| 파라미터 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| `problem_id` | number | **필수** | BOJ 문제 번호 (예: 1000) |

### 출력 스키마

```typescript
{
  problemId: number;
  title: string;            // 한글 제목
  titleEn?: string;         // 영문 제목 (있는 경우)
  level: number;            // 1-30
  tierName: string;         // "Gold III"
  tierBadge: string;        // "🟡 Gold III"
  tags: [
    {
      key: string;          // "dp"
      name: string;         // "다이나믹 프로그래밍"
      problemCount: number; // 2847
    }
  ];
  acceptedUserCount: number;
  averageTries: number;
  bojLink: string;          // https://www.acmicpc.net/problem/1927
  isSolvable: boolean;
  isOfficial: boolean;
}
```

### 사용 예시

#### 예시 1: 문제 정보 조회
**사용자**: "1927번 문제에 대해 알려줘"

**도구 호출**:
```json
{
  "problem_id": 1927
}
```

**응답**:
```
[1927] 최소 힙

📊 난이도: 🟤 Silver II (레벨 9)

🏷️ 태그:
- 자료 구조 (2,847개 문제)
- 우선순위 큐 (456개 문제)

📈 통계:
- 73,425명이 해결
- 평균 1.8회 시도

🔗 문제 링크: https://www.acmicpc.net/problem/1927
```

#### 예시 2: 복수 문제 비교
**사용자**: "1920번과 1654번 문제 중 어떤 게 더 어려워?"

**도구 호출** (2회):
```json
{ "problem_id": 1920 }
{ "problem_id": 1654 }
```

**응답**:
```
두 문제를 비교해드리겠습니다:

[1920] 수 찾기 (Silver IV, 레벨 7)
- 평균 1.8회 시도
- 89,234명 해결

[1654] 랜선 자르기 (Silver II, 레벨 9)
- 평균 2.4회 시도
- 67,892명 해결

1654번이 2단계 더 어렵습니다 (레벨 7 vs 9).
```

### 주의사항

- 존재하지 않는 문제 번호는 에러 반환
- 문제 번호는 양의 정수여야 함
- 응답 데이터는 1시간 캐싱됨 (최신 통계는 약간 지연 가능)

### 에러 케이스

| 에러 | 원인 | 해결 방법 |
|------|------|----------|
| "Problem not found" | 존재하지 않는 문제 ID | BOJ에서 문제 번호 확인 |
| "Invalid problem ID" | 음수 또는 0 입력 | 양의 정수 입력 |

```typescript

```

| 파라미터 | 타입 | 필수 | 설명 |

## get_hint

### 설명
AI 기반으로 문제에 대한 단계별 힌트를 생성합니다.
3단계 힌트 시스템을 통해 점진적으로 문제 해결을 도와줍니다.

### 입력 스키마

```typescript
{
  problem_id: number;       // BOJ 문제 번호 (필수)
  hint_level: 1 | 2 | 3;    // 힌트 레벨 (필수)
  user_context?: string;    // 사용자의 현재 이해도 (선택)
}
```

#### 파라미터 상세

| 파라미터 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| `problem_id` | number | **필수** | BOJ 문제 번호 |
| `hint_level` | 1\|2\|3 | **필수** | 힌트 레벨 (1=기본, 2=중급, 3=상세) |
| `user_context` | string | 선택 | 사용자의 현재 이해도 또는 막힌 부분 |

#### 힌트 레벨 가이드

| 레벨 | 내용 | 적합한 상황 |
|------|------|-----------|
| **1** | 문제 패턴 인식 (어떤 알고리즘인지) | 문제 접근 방향을 모를 때 |
| **2** | 핵심 통찰 및 접근법 (완전한 해법 제외) | 알고리즘은 알지만 구체적 방법 모를 때 |
| **3** | 상세한 알고리즘 단계 (코드는 제외) | 거의 다 이해했지만 세부 단계 확인 필요할 때 |

### 출력 스키마

```typescript
{
  problemId: number;
  problemTitle: string;
  hintLevel: number;
  hint: string;             // 마크다운 형식의 힌트
  suggestedNextLevel?: number;  // 다음 힌트 레벨 제안
  relatedConcepts?: string[];   // 관련 개념
}
```

### 사용 예시

#### 예시 1: 레벨 1 힌트 (문제 패턴 인식)
**사용자**: "11053번 문제 어떻게 풀어야 할지 모르겠어. 힌트 줘"

**도구 호출**:
```json
{
  "problem_id": 11053,
  "hint_level": 1
}
```

**응답**:
```
[11053] 가장 긴 증가하는 부분 수열 - 레벨 1 힌트

🎯 문제 패턴

이 문제는 **동적 계획법(Dynamic Programming)** 문제입니다.

부분 수열의 각 위치에서 "이전 원소들을 고려했을 때
여기까지의 최선의 선택"을 저장하는 방식으로 접근해보세요.

작은 부분 문제의 해를 저장하고, 이를 조합하여
더 큰 문제를 해결하는 방식을 고민해보세요.

💡 다음 단계
레벨 2 힌트에서는 DP 배열을 어떻게 정의하고
점화식을 세우는지 알려드리겠습니다.
```

#### 예시 2: 레벨 2 힌트 (핵심 통찰)
**사용자**: "DP라는 건 알겠는데 배열을 어떻게 정의해야 할지 모르겠어. 레벨 2 힌트 줘"

**도구 호출**:
```json
{
  "problem_id": 11053,
  "hint_level": 2,
  "user_context": "DP라는 것은 알지만 상태 정의를 모르겠음"
}
```

**응답**:
```
[11053] 가장 긴 증가하는 부분 수열 - 레벨 2 힌트

🧩 핵심 아이디어

DP 배열을 다음과 같이 정의해봅시다:
**dp[i] = i번째 원소를 마지막으로 하는 가장 긴 증가 부분 수열의 길이**

각 위치 i에서, 이전 위치들(j < i)을 확인하며:
- arr[j] < arr[i]인 경우 (증가 조건 만족)
- dp[j]가 최대인 것을 찾습니다

점화식:
```
dp[i] = max(dp[j]) + 1 (단, j < i이고 arr[j] < arr[i])
```

초기값: dp[0] = 1 (첫 번째 원소 자체가 길이 1인 수열)

최종 답: max(dp[0], dp[1], ..., dp[n-1])

💡 다음 단계
레벨 3 힌트에서는 구체적인 알고리즘 단계와
시간 복잡도까지 설명드리겠습니다.
```

#### 예시 3: 레벨 3 힌트 (상세 알고리즘)
**사용자**: "레벨 3 힌트까지 보고 싶어"

**도구 호출**:
```json
{
  "problem_id": 11053,
  "hint_level": 3
}
```

**응답**:
```
[11053] 가장 긴 증가하는 부분 수열 - 레벨 3 힌트

📋 알고리즘 단계

**1단계: 초기화**
- n = 수열의 길이
- arr[] = 입력 수열
- dp[] = [1, 1, 1, ..., 1] (길이 n, 모두 1로 초기화)

**2단계: DP 배열 채우기**
```
for i in range(1, n):
    for j in range(0, i):
        if arr[j] < arr[i]:
            dp[i] = max(dp[i], dp[j] + 1)
```

각 i에 대해 이전 모든 j를 확인하며,
증가 조건을 만족하는 경우 dp 값을 갱신합니다.

**3단계: 최댓값 찾기**
```
answer = max(dp)
```

⏱️ 시간 복잡도: O(n²)
- 외부 루프: n번
- 내부 루프: i번 (평균 n/2번)
- 전체: n × n/2 = O(n²)

💾 공간 복잡도: O(n)
- dp 배열: n개 원소

🎓 관련 개념
- 동적 계획법 (Dynamic Programming)
- 최장 증가 부분 수열 (LIS)
- 더 빠른 풀이: O(n log n) 이분 탐색 활용

이제 직접 코드로 구현해보세요!
```

### 주의사항

- 레벨 3 힌트도 **실제 코드는 제공하지 않습니다**
- 힌트는 문제의 태그와 난이도를 기반으로 생성됩니다
- `user_context`를 제공하면 더 맞춤화된 힌트를 받을 수 있습니다
- 힌트 생성에는 2-5초 소요될 수 있습니다 (AI 처리 시간)
