## 핵심 가치 제안

### 헤드라인
**백준 알고리즘 학습을 AI와 함께, 스마트하게**

### 서브 헤드라인
Claude Code와 함께 문제를 분석하고, 단계별 힌트를 받고, 복습 문서를 자동으로 생성하세요.
설정 불필요, API 키 불필요. 설치 후 바로 시작.

### 해결하는 문제

백준 온라인 저지로 알고리즘을 공부하면서 이런 고민 하셨나요?

- 💭 **"이 문제, 어디서부터 시작해야 할지 막막해..."**
  → 3단계 힌트 시스템으로 단계적 학습

- 📝 **"풀고 나면 정리는 귀찮고, 나중에 보면 기억이 안 나..."**
  → AI가 복습 문서 템플릿 자동 생성

- 🔍 **"내 수준에 맞는 문제를 찾기가 힘들어..."**
  → 난이도, 알고리즘별 스마트 검색

- ⚙️ **"API 설정부터 막히면 시작도 못해..."**
  → Zero Configuration, 설치 후 바로 사용

**algokit**는 이 모든 문제를 해결하는 백준 학습 도우미입니다.

## 주요 기능

### 1️⃣ 문제 분석 및 단계적 힌트

```
사용자: "11053번 가장 긴 증가하는 부분 수열 문제 분석해줘"

Claude: "이 문제는 동적 프로그래밍(DP) 문제입니다.
        Gold II 난이도로, 상위 40-60% 수준입니다."

사용자: "힌트 좀 줘"

Claude: [Level 1 - 문제 분석]
        "이 문제는 최적 부분 구조를 가지고 있습니다.
         dp[i]의 의미를 먼저 정의해보세요..."

사용자: "더 자세히 알려줘"

Claude: [Level 2 - 핵심 아이디어]
        "dp[i] = i번째 원소를 마지막으로 하는 가장 긴 증가 수열의 길이
         점화식: dp[i] = max(dp[j]) + 1 (j < i, arr[j] < arr[i])"
```

**특징**:
- 🎯 **문제별 맞춤 힌트**: 일반적인 DP 설명이 아닌, 이 문제에 특화된 힌트
- 📊 **3단계 학습**: 문제 분석 → 핵심 아이디어 → 상세 풀이
- 🤖 **자동 레벨 선택**: Claude Code가 사용자 상황 분석 후 적절한 힌트만 제공
- ⚡ **즉시 응답**: 500ms 이내 프롬프트 생성

### 2️⃣ 복습 문서 자동 생성

```
사용자: "1927번 최소 힙 문제 복습 문서 만들어줘"

Claude: "복습 문서를 작성하겠습니다. 몇 가지 질문드릴게요.

        1. 어떤 접근법으로 해결하셨나요?"

사용자: "Python heapq 모듈 사용했어요.
        0 입력 시 최솟값 출력하고 제거했습니다."

Claude: "2. 시간/공간 복잡도는 어떻게 되나요?"

사용자: "시간 O(N log N), 공간 O(N)"

Claude: "완벽합니다! 복습 문서를 생성했습니다 📄

        [마크다운 문서 출력]

        관련 문제도 추천드립니다:
        - [11279] 최대 힙 (⚪ Silver II)
        - [11286] 절댓값 힙 (⚪ Silver I)"
```

**특징**:
- 📝 **대화형 작성**: AI가 질문하고, 사용자 답변을 템플릿에 채워넣기
- 🔗 **관련 문제 추천**: 같은 알고리즘, 비슷한 난이도 문제 자동 추천
- 📅 **자동 메타데이터**: 문제 정보, 태그, 난이도 자동 포함
- 💾 **즉시 저장**: 마크다운 형식으로 바로 저장 가능

### 5️⃣ 코드 분석 및 피드백 (Phase 6)

```
사용자: "내 코드 분석해줘"

[Python 코드 붙여넣기]

Claude: "코드를 분석했습니다.

        ✅ 장점:
        - 알고리즘 선택 적절 (BFS)
        - 시간복잡도 O(V+E) 효율적

        💡 개선점:
        1. 변수명 명확화
           arr → graph (그래프 의미 명확)

        2. 자료구조 최적화
           list → deque (popleft O(1))

        3. 예외 처리 추가
           빈 그래프 케이스 처리 필요

        개선된 코드를 보여드릴까요?"
```

**특징**:
- 🔍 **자동 분석**: 알고리즘, 복잡도, 자료구조 선택 검토
- 💬 **구체적 피드백**: "잘했어요"가 아닌, 실행 가능한 개선안
- 🎓 **학습 강화**: 왜 이 방법이 더 나은지 설명

## 사용자 시나리오

### 시나리오 1: 초보자의 첫 DP 문제 도전

**주인공**: 대학생 지민 (Silver 2 티어)
**상황**: DP 문제를 처음 접했지만 어디서부터 시작해야 할지 모름

```
지민: "Claude, 1463번 1로 만들기 문제 분석해줘"

Claude: "이 문제는 동적 프로그래밍 입문 문제입니다.
        Silver III 난이도로, 초급 (상위 70-80%) 수준이에요."

지민: "DP가 뭔지도 잘 모르겠어... 힌트 좀 줘"

Claude: [Level 1 - 문제 분석]
        "DP는 큰 문제를 작은 문제로 나누어 해결하는 방법입니다.
         이 문제에서는 dp[i]를 'i를 1로 만드는 최소 연산 횟수'로 정의해보세요.

         예를 들어:
         - dp[1] = 0 (이미 1이므로)
         - dp[2] = 1 (2 → 1, 1번 연산)
         - dp[3] = 1 (3 → 1, 1번 연산)"

지민: "아하! 그럼 dp[10]은 어떻게 구하지?"

Claude: [Level 2 - 핵심 아이디어]
        "dp[10]을 구하려면 이전 값들을 활용합니다:

         1. 10 → 9 (1 빼기): dp[9] + 1
         2. 10 → 5 (2로 나누기): dp[5] + 1

         이 중 최솟값을 선택하면 됩니다.
         점화식: dp[i] = min(dp[i-1], dp[i/2], dp[i/3]) + 1"

지민: "이제 코딩할 수 있을 것 같아!"

[30분 후]

지민: "문제 풀었어! 복습 문서 만들어줘"

Claude: "축하합니다! 복습 문서를 작성하겠습니다.
        어떤 방법으로 구현하셨나요?"

[대화 후 복습 문서 자동 생성]
```

**결과**:
- ✅ DP 개념 이해
- ✅ 첫 DP 문제 해결
- ✅ 복습 문서 작성으로 장기 기억 강화

## 차별화 포인트

### 🆚 기존 방법 vs algokit

| 기존 방법 | algokit |
|---------|----------|
| 구글링 → 블로그 찾기 → 산발적 정보 | AI와 대화하며 맞춤형 힌트 |
| 힌트 = 정답 코드 (학습 효과 ↓) | 3단계 힌트로 단계적 학습 |
| 복습 문서 수동 작성 (귀찮음) | 대화형 자동 생성 |
| 문제 찾기 = 백준 사이트 검색 | Claude 내에서 스마트 검색 |
| 코드 리뷰 = 혼자 고민 | AI 피드백 + 개선안 제시 |

### ⚡ 프롬프트 기반 아키텍처

**기술적 혁신**:

```
기존 방법 (LLM 기반):
User → MCP Server → LLM API (2-5초, 비용 발생) → User

algokit (프롬프트 기반):
User → MCP Server (프롬프트 생성, < 500ms) → Claude Code (LLM) → User
```

**장점**:
1. **문제 특화**: 모든 DP 문제 동일 힌트 → 문제별 맞춤 힌트
2. **빠른 응답**: 500ms 이내 프롬프트 생성
3. **비용 절감**: MCP 서버 API 비용 0원
4. **확장성**: 새 알고리즘 추가 시 1곳만 수정

**코드 규모**:
- Phase 3 (하드코딩): 1,834줄
- Phase 5 (프롬프트): 570줄 (-69%)

// 프롬프트 구조 검증

## 기술 스펙

### 아키텍처

```
사용자 (User)
    ↓
Claude Code (AI Client)
    ↓ MCP Protocol
algokit Server
    ├─ 문제 분석 (analyze_problem)
    ├─ 힌트 생성 (generate_hint)
    ├─ 복습 템플릿 (generate_review_template)
    ├─ 문제 검색 (search_problems)
    ├─ 문제 조회 (get_problem)
    ├─ 태그 검색 (search_tags)
    ├─ 본문 스크래핑 (fetch_problem_content)
    └─ 코드 분석 (analyze_code_submission)
    ↓
solved.ac API (무료, 인증 불필요)
```

## FAQ

### Q1. API 키가 정말 필요 없나요?
**A**: 네! algokit 서버 자체는 API 키가 필요 없습니다. solved.ac API는 무료이고 인증이 필요 없습니다. Claude Code/Desktop은 별도 계정이 필요하지만, 이는 algokit와 무관합니다.

### Q2. 비용이 발생하나요?
**A**: algokit 서버 사용은 무료입니다. Claude Desktop 무료 플랜 또는 Claude API를 사용하시면 됩니다.

### Q3. 힌트가 너무 자세하면 학습 효과가 떨어지지 않나요?
**A**: 3단계 힌트 시스템으로 해결했습니다. Claude Code가 사용자 상황을 분석해서 적절한 1개 레벨만 제공합니다. Level 1은 방향성만, Level 2는 핵심 아이디어, Level 3는 상세 풀이입니다.

### Q4. 어떤 문제까지 지원하나요?
**A**: solved.ac API에 등록된 모든 백준 문제를 지원합니다 (약 28,000+개). Bronze부터 Ruby까지 모든 난이도를 지원합니다.

### Q5. 다른 코딩 테스트 플랫폼(프로그래머스, 리트코드 등)도 지원하나요?
**A**: 현재는 백준(BOJ)만 지원합니다. 다른 플랫폼은 향후 검토 예정입니다.

### Q6. 문제 본문을 직접 보여주나요?
**A**: Phase 6 기능으로 `fetch_problem_content` 도구를 통해 문제 본문을 스크래핑해서 보여줍니다. 캐싱을 지원하여 빠른 조회가 가능합니다.

### Q7. 내 백준 계정과 연동되나요?
**A**: Phase 7에서 백준 ID 기반 학습 분석 기능을 추가할 예정입니다. 현재는 문제 정보 조회만 가능합니다.
